The problem

Everything downstream of a match trusts a single API's word.

Settlement, prediction markets, parametric insurance, fan rewards — all of it depends on "what actually happened on the pitch". A single data source is a single point of trust. Existing oracle designs fix this with heavy machinery: staking, tokens, dispute courts, governance votes.

AgentVAR replaces the machinery with one lightweight primitive: the payment itself is the incentive.

// the old way
trust(oneSportsApi.result) → pray
stake → slash → dispute → govern → wait…

// the AgentVAR way
pay per testimony — 3 agents, 3 sources
truth pays · lies don't get the sale
ruling → contract → money moves
How a ruling happens

From goal to on-chain truth in five moves.

01 / SCOUT
📡

A review opens

The Scout agent watches the match feed — a real recorded fixture, or a live 2026 World Cup feed — and opens a VAR review for every key event: "Did Messi score at 83'?"

02 / JURY
✍️

Jurors testify blind

Three juror agents each pull raw evidence from their own source, reason over it with an LLM, and return a signed testimony. None sees another's answer.

03 / ARBITER
⚖️

Tally & cross-examine

The Arbiter verifies signatures, tallies a 2/3 majority, and cross-examines any dissenter with the majority's evidence before ruling.

04 / x402
💸

Truth gets paid

Majority jurors receive a real USDC micropayment via x402. The dissenter's claim endpoint is simply never called — the withheld fee is the whole punishment.

05 / CHAIN
⛓

Money moves itself

The ruling anchors on TruthOracle; the ParametricPool contract re-verifies it on-chain and pays out instantly. No claims process exists.

The crew

Four autonomous agents. One economy where honesty is the only profitable strategy.

📡

Scout

match watcher

Monitors the live feed and decides which moments deserve adjudication. Opens reviews; never votes.

✍️

Jurors ×3

paid witnesses

Each juror has its own data source — in live mode, real independent providers (ESPN, TheSportsDB, football-data.org). Each reasons independently and signs its testimony with its own key.

⚖️

Arbiter

the referee

Collects testimony, rules by 2/3 majority, cross-examines dissent, announces the verdict stadium-style — then settles who gets paid and who doesn't.

🏦

Treasurer

pool caller

Watches rulings for the parametric term. It can only ask the pool to pay — the contract re-verifies the ruling on-chain itself.

The economics

An honest-truth supply chain, every hop metered by x402.

+0.05 USDC
A consumer (AI assistant via MCP, or any app) pays the Arbiter for one adjudicated ruling.
−0.01 × majority
The Arbiter pays each juror whose testimony matched the ruling, via that juror's x402-gated claim endpoint.
0.00 for liars
Dissenting testimony is recorded on-chain and its fee withheld. No slashing, no tribunal — the lie just didn't sell.
The Arbiter's margin is the spread. A juror's reputation is its revenue stream — and every receipt is a real transaction on Injective EVM testnet.
Injective technology

All four new Injective technologies, used where they carry weight.

x402

The consensus incentive layer

Every endpoint is gated by @injectivelabs/x402 middleware with per-route payTo — testimony fees flow to each juror's own wallet, adjudication fees to the arbiter. Payment = accepted; withheld payment = the entire punishment mechanism. Settles as native USDC in ~650ms.

USDC · CCTP

Cross-chain pool funding

The ParametricPool holds native Circle USDC (domain 29). npm run cctp burns USDC on Ethereum Sepolia via TokenMessengerV2, fetches Circle's Iris attestation, and mints directly into the pool on Injective — sponsors on any chain can capitalize World Cup insurance.

MCP Server

Truth as a tool call

Two MCP roles: a consumer server (adjudicate_event, list_rulings, juror_leaderboard) so any AI assistant buys jury-verified truth — with a real x402 payment — and a juror role that makes each juror its own MCP endpoint.

Build with it

Four ways in — from a browser tab to your own settlement pipeline.

TypeScript SDK

AgentVARClient — truth for your app

The sdk/ client pays x402 fees automatically (402 → sign USDC transfer → retry) and verifies every juror's Ed25519 signature locally, so you never have to take the server's word for who said what. Built for prediction markets, betting apps and parametric insurance.

const agentVar = new AgentVARClient({ privateKey });
const ruling = await agentVar.adjudicate(
  { team: "Argentina", minute: 83 }   // 0.05 USDC via x402
);
agentVar.verifyTestimony(t)           // ✅ local, trustless

npm run sdk:example settles a demo prediction market end-to-end.

MCP · HTTP · Live mode

For AI assistants & everything else

MCP: attach npm run mcp to Cursor/Claude and your assistant buys adjudicated truth as a tool call.

HTTP: POST /api/adjudicate — one x402-gated endpoint, any language.

Live real-match mode: MATCH_MODE=live points the jurors at real, independent providers (ESPN / TheSportsDB / football-data.org) for the actual 2026 World Cup; AUTO_ADJUDICATE=true rules on new goals the moment they land — no clicks.

Run it yourself

The crew runs on your machine, with your keys. That's the point.

AgentVAR is self-hosted by design — portable, inspectable, no trusted operator. This site is fully static: the intro you're reading, plus an observatory that reads the contracts straight from Injective. Everything interactive runs locally: 

git clone https://github.com/Qinsir7/AgentVAR && cd AgentVAR && npm install

# 30-second demo (no wallet, simulated payments)
npm start                        # → localhost:4402/dashboard

# Full on-chain mode — bring ONE funded testnet key (.env → FUNDER_PRIVATE_KEY)
npm run setup && npm run deploy  # wallets, contracts, pool funding: automatic
PAYMENT_RAIL=injective npm start # x402 + anchors + payouts, all real

# Watch a real World Cup match, hands-free
npm run start:live
